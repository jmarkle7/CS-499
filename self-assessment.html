Professional Self-Assessment

Finishing my Computer Science program at Southern New Hampshire University has allowed me to learn new skills, both technically and professionally that will help me in my career. At the beginning of my program, I had zero knowledge about how to code or how systems and software actually work. Coming towards the end of my degree, I can create functional software, create and manage databases, design mobile applications, and use algorithms. CS 340 was a class that taught me skills about how to communicate with a client and build software to fit their needs. This class showed me how important communication is between development teams, testing teams, clients, and stakeholders because they all have a part during development. My ePortfolio also contains three artifacts with different projects that I created throughout my degree and enhanced these artifacts to showcase my skills. Specifically, you will be able to see how I demonstrated software design and engineering, algorithms and data structure, and databases as I enhanced my artifacts.

One thing that I learned about the field of Computer Science during my program was the importance of collaborating in a team environment. Team members from the development side must communicate with the client and stakeholders to effectively create software to fit their needs. I also learned about different development phases such as Agile and Scrum teams work and how each differs from each other. CS 250 was a course that really educated me on what steps teams take for success versus what problems can occur during the Software Development Lifecycle. CS 499 has also played a role in my communication skills because I created a code review which goes in depth about how my software works and what enhancement plans I had for my portfolio.

Another skill I learned during my course is Data Structures and Algorithms. Before I began my degree, I didn’t understand how data was stored or how it was actually used in software. After taking classes like CS 260, I was able to understand that data is extremely important and can impact scalability, and response times for software. One project that helped me understand this skill was creating my Weight Tracking Mobile Application in CS 360. Data structures and algorithms were important because it allowed users to be able to create/login to their accounts and store their weight logs. Without proper data structures and algorithms, software can be less functional and usable for a user. Testing was also a necessary skill because most of the time it’s not going to be perfect the first time, however; testing for all use cases led to proper implementation.

Learning software engineering and database skills has also been helpful for advancement in my career. During my CS 465 course, I developed a full stack application showcasing different travel packages for a client. Creating this project involves creating a user interface, creating functions, handling data, and API communication. Overall, this project really helped me understand how each component of a full stack application works together from front end to backend. Databases have also been a huge part in my Computer Science degree while working with them in numerous classes. I learned that data can be used to analyze and make company decisions or be used to store user information. Overall, these skills have allowed me to be able to create software from scratch and be able to handle data effectively.

Having a security mindset during my program has also shaped my skills because I want to ensure that software is safe for clients and users. CS 305 was an important course for learning the risks of low security software and how attackers can get access to important files such as company data, passwords, user data, etc. Troubleshooting software for vulnerabilities and making sure libraries or other components are up to date is one skill I learned. I also learned throughout this course that security isn’t a specific stage in the development lifecycle, but it should be considered from start to beginning to ensure clients and users are safe. As I continue my career outside of SNHU, I will keep a security mindset in the workforce.

The artifacts chosen for my ePortfolio include a Weight Tracking Mobile Application, and a Python Crud Dashboard. These artifacts are showcased to demonstrate my skills in software engineering and design, algorithms and data structures, and databases. My Weight Tracking Mobile application showcases how I can create new screens, progress features, goal tracking, data organization, and specific calculation. My Python Crud Dashboard demonstrates my ability to create filters to easily access across 10,000 records from an animal shelter data set. This portfolio showcases my growth throughout my Computer Science program and specific technical skills I have gained.
